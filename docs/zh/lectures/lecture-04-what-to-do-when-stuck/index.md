每个研究循环最终都会停滞。指标停止改进，agent 一次又一次地尝试变体却毫无进展。如果循环对此没有应对策略，它将把剩余的全部预算浪费在死胡同的细微调整上。

autoresearch 框架为此提供了**三级转向策略**。当 agent 连续 N 次迭代未能改进指标时，卡住检测机制触发：连续3次未改进 → 一级转向（L1），连续5次未改进 → 二级转向（L2），预算耗尽 → 三级终止（L3）。**L1 转向**是保守的：在当前范式内换个方向，例如从优化某一排序算法族切换到完全不同的算法族。**L2 转向**是范式转移：质疑当前方法的根本假设，例如从"让排序更快"转向"是否可以完全避免排序"。在转向之前，agent 会执行"更深层思考"：重读完整历史，找出最佳3次和最差3次实验的共同规律，生成5个尚未尝试的假设并按预期影响力排序。**L3** 在 `max_iterations` 耗尽后触发，agent 撰写 `final_report.md`，内容包括最佳结果、成功路径总结、失败原因以及下次会话的推荐假设——成为下一个会话的起点。

噪声指标（如 LLM 评分或网络延迟）会导致卡住检测过早触发。解决方案是在 `research.md` 中设置 `noise_runs: 3` 或更高，并配合 `min_delta` 过滤掉噪声范围内的"改进"，确保只有真实的进步才会被计入。

## 核心概念

**卡住检测（stuck detection）**：当连续 N 次迭代未改进指标时自动触发的机制，阈值在 `research.md` 的 Constraints 字段中配置。

**L1 转向（策略切换）**：在当前范式内切换到不同的策略方向，触发阈值为3次连续未改进。

**L2 转向（范式转移）**：质疑当前方法的根本假设，寻找全新的问题框架，触发阈值为5次连续未改进。

**"更深层思考"规则**：在转向之前进行结构化历史分析——寻找模式、生成候选假设、按影响力排序——而非随机探索。

**`noise_runs` 与 `min_delta`**：用于降低噪声指标方差的配置参数，防止因测量噪声误判为停滞。

## 关键要点

- 卡住检测是主动的——不要等预算耗尽才换策略
- L1 转向（3次迭代）：在当前范式内改变方向
- L2 转向（5次迭代）：质疑范式本身
- L3（`max_iterations` 耗尽）：写 `final_report.md`，下一个会话从它开始
- 噪声指标导致错误的卡住检测——使用 `noise_runs` 和 `min_delta` 应对
- "更深层思考"意味着对历史规律的结构性反思，而非随机探索
- 历史中的"接近成功"实验往往是突破性尝试的种子
