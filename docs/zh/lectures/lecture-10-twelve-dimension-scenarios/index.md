功能通常只为"快乐路径"而设计。`/autoresearch:scenario` 系统性且自主地探索完整的场景空间——涵盖12个情境多样性维度。

12个维度覆盖了任何功能或系统可能面临的情境全集：(1) 正常路径、(2) 错误条件、(3) 边界情况、(4) 滥用情况、(5) 规模压力、(6) 并发、(7) 时间性、(8) 数据变体、(9) 权限差异、(10) 第三方集成、(11) 恢复场景、(12) 状态转换。场景循环每次迭代跨这12个维度生成**一个**情境，确保系统性全面覆盖，而非随机抽样。不同领域有不同的维度优先级：软件领域重点关注错误条件、并发、规模和恢复；安全领域重点关注滥用情况、权限、数据变体和集成；产品领域重点关注正常路径、用户故事、权限和时间性。

场景输出可适配多种下游格式：`--format test-scenarios`（生成带前置条件、步骤和预期结果的 pytest/jest 测试用例）、`--format user-stories`（生成敏捷用户故事）、`--format use-cases`（生成正式用例文档）、`--format threat-scenarios`（生成安全威胁场景）。通过 `--chain debug` 可对发现的边界情况直接进行 bug 调查；通过 `--chain security` 可对发现的威胁场景进行安全审计。12维扫描能捕捉到手动场景编写几乎不会考虑到的情况——例如"两个用户同时结账最后一件商品"这样需要主动思考"并发"维度才能发现的竞态条件。

## 核心概念

**12维度框架**：系统性情境分类法，确保场景探索覆盖正常路径之外的所有重要情境类别，包括边界情况、并发、滥用、恢复等。

**每次迭代一个场景**：有条不紊而非随机——每次迭代专注于一个维度生成一个场景，使覆盖情况可追踪。

**领域特定维度优先级**：不同领域（软件/安全/产品/业务）对12个维度有不同的优先权重，聚焦于最相关的覆盖方向。

**输出格式适配**：同一批场景可渲染为测试用例、用户故事、用例文档或威胁场景，适配不同下游需求。

**`--chain` 链式命令**：将生成的场景直接导入 `debug` 或 `security` 命令，构成完整质量流水线。

## 关键要点

- 12个维度提供对场景空间的系统性全面覆盖
- 每次迭代一个场景——有条不紊，而非随机
- 输出格式决定下游用途（测试、故事、威胁）
- 领域特定维度优先级将覆盖方向调整至最重要的方面
- 与 debug 和 security 链接，构建完整质量流水线
- 12维扫描能捕捉到手动场景编写几乎不会覆盖的情况
