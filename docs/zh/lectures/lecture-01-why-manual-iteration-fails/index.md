2025 年，Andrej Karpathy 发布了一个 630 行的 Python 脚本，能在一夜之间无人值守地运行 100 次 ML 实验——自动回滚失败，将每次改进叠加积累。结果：经验丰富的研究员需要数周调优的模型，在 48 小时的过夜运行中被超越。

让这成为可能的不是更聪明的模型，而是更聪明的**循环**。

## 手动迭代的问题

大多数研究员的迭代方式是：运行实验、等待、分析结果、形成假设、做出改变、重复。顺利时每天 3–5 次实验，通常只有 1–2 次。每次实验都需要人的注意力：读输出、决定下一步、手动回滚失败。

这种方式有三个致命缺陷：

**人的注意力是瓶颈。** 你在睡觉、开会或做其他事情时无法运行实验。研究循环被你的可用时间所限制。

**手动回滚慢且有损。** 当实验让情况变糟时，回滚是手工劳动——研究员往往根本不回滚，让坏状态在代码库中积累。

**主观判断悄然渗入。** "这看起来有希望"不是指标。没有机械评估，研究员会保留感觉良好但实际上没有改进目标的实验。

## Karpathy 的洞见

Karpathy 的 autoresearch 脚本体现了三个改变一切的原则：

**一个指标，一个方向。** `val_bpb`（验证集每字节比特数，越低越好）。不是"模型质量"或"训练稳定性"——一个数字告诉 agent 该优化什么。每个决策都归结为：这个改变让 `val_bpb` 降低了吗？

**受约束的范围。** agent 只能修改 `train.py`，不能触碰数据准备、分词器或评估代码。这意味着每次失败都是可隔离的：如果指标变差，原因一定在 `train.py` 里。

**机械验证 + 自动回滚。** 每次改变后，脚本运行训练恰好 5 分钟并测量指标。如果改进：`git commit` 保留。如果变差：`git revert` 尝试下一个。不需要人的判断，不需要手动清理。

结果是一个**每晚都在复利**的系统。每次迭代建立在最佳前一状态之上。失败被自动遗忘。成功被永久记录在 git 历史中。

## 核心概念

**自主研究循环**：一个自我运行的 修改→验证→保留/丢弃→重复 周期，迭代之间无需人工干预。

**机械指标**：可以通过定义的程序自动计算的数字，有明确的方向（高或低更好）。

**保留策略（keep policy）**：决定保留还是丢弃改变的规则。最常见：`score_improvement`（新分数优于前最佳则保留）。

**自动回滚**：当改变使指标变差时，`git revert` 自动恢复上一个状态。失败的实验保存在 git 历史中，但不影响代码库。

**有界 vs 无界模式**：有界循环运行恰好 N 次迭代后停止。无界循环运行直到达到目标指标或用户中断。

## 关键要点

- 手动迭代被人的注意力所限制——agent 完全消除了这个瓶颈
- 三个原则解锁了自主研究：一个指标、受约束的范围、机械验证
- 自动回滚意味着失败是免费的——agent 可以大胆尝试，因为坏的改变不会造成损失
- 任何有可测量指标的领域都可以使用 autoresearch 循环
- 复利效应是真正的力量：每晚在前一晚的最佳结果之上构建
