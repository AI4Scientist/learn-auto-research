autoresearch 循环在有机械指标时运行良好。但对于没有客观指标的决策怎么办？架构选择、产品策略、内容质量、设计权衡——这些本质上是主观的。`/autoresearch:reason` 通过**盲审裁判团**作为适应度函数，将 autoresearch 循环扩展到主观领域。裁判团就是指标：它是决策领域的 `val_bpb`。

精炼循环分为六步：**生成候选A** → **批评者攻击**（冷启动，未看过生成过程，尽可能猛烈地攻击候选A）→ **作者B回应**（冷启动，只看A和批评，生成改进后的候选B）→ **合成**（将A和B的最强元素合并为候选C）→ **盲审裁判团**（N个裁判在不知道哪个是"新提案"哪个是"现有方案"的情况下，只看内容选出较好的一个）→ **更新并重复**（获胜候选成为新的A，循环直到连续N次获胜或达到 `max_iterations`）。**关键不变量**：每个 agent 都是冷启动的全新调用，不看辩论历史，防止相关性偏差（agent 为了与之前说的保持一致而改变评估）。

盲审之所以有效，是因为有色眼镜式的评审不可靠——裁判知道哪个选项是"新提案"时，会带入锚定偏差、损失厌恶和社会压力。盲审移除了这一切：裁判只看 X 和 Y，基于内容本身做出选择。收敛后生成的 `handoff.json` 包含胜出立场及支撑证据，可被任何下游命令读取。常用链式模式：`reason → plan`（收敛后脚手架研究项目进行实证验证）、`reason → predict`（收敛后用五专家分析进行压力测试）。

## 核心概念

**主观决策的指标替代**：当没有机械指标时，盲审裁判团充当适应度函数——裁判的多数判断就是"分数"。

**冷启动 agent**：每个参与辩论的 agent 都是全新调用，不看历史，防止相关性偏差和群体思维。

**盲审（blind judging）**：裁判不知道哪个选项是"新的"、哪个是"旧的"，只基于内容质量做出选择，排除权威偏差和锚定效应。

**收敛条件**：候选在连续 `--convergence N`（默认3）轮中获胜，循环停止。

**`handoff.json`**：收敛时生成的结构化摘要，包含胜出立场、裁判一致性和推荐下一步，可被任何下游命令读取。

## 关键要点

- 当没有指标时，盲审裁判团就是指标
- 冷启动 agent 防止相关性偏差和群体思维
- 盲审移除权威偏差和锚定效应——纯内容胜出
- 收敛（连续N次获胜）是停止条件
- `handoff.json` 将输出链到任意下游命令
- 对于架构决策，使用 `reason → plan → autoresearch` 链进行实证验证
